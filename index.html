    使用文档

    debug.js 为未压缩版本 min.js 为压缩后的版本

    由于项目使用了 async 和 await，在有 regeneratorRuntime 的环境中或者自行添加了 regeneratorRuntime 的 polyfill，可以使用 need-regenerator 的版本，其它情况下需使用 with-regenerator 的版本。

    引用示例

    脚本可动态引入、加载。在加载完成前后，均可使用。

    
tnpm install @ali/lib-js-sdk
import '@ali/lib-js-sdk';

    加载完成前使用

    如需在库加载完成前使用，有两种使用方法

    1. 使用全局数组，往全局数组 libSQ 中 push 命令和参数;
    2. 使用document 的 ready 事件监听，事件名为 lib-sq-ready;
    <script>
// 使用全局数组，往全局数组 `libSQ` 中 `push` 命令和参数;

// 使用数组形式调用的示例
window.libSQ = window.libSQ || [];
window.libSQ.push([
  'get-user-info', // 命令名，见 API 文档
  // 命令参数
  {
     // 成功时回调
     success: function(info) {
      alert('[from push][success] ' + JSON.stringify(info, null, 2));
    },
    // 失败时回调
    error: function(err) {
      alert('[from push][error] ' + JSON.stringify(err, null, 2));
    },
    // 透传至底层调用的参数
    params: {
      isRefresh: '1',
    },
  }
]);

// 使用`document` 的 ready 事件监听，事件名为 `lib-sq-ready`;

// 通过 ready 事件使用 JS SDK 的示例，如不传入 `success`、`error`，则以 `Promise` 形式处理。
// 推荐使用 `Promise` 形式。
document.addEventListener('lib-sq-ready', (evt) => {
  const {lib} = evt.detail;

  // 无参数直接调用示例
  lib.exec('get-app-info').then(function (info) {
    alert('[from ready][get-app-info][then] ' + JSON.stringify(info, null, 2));
  }).catch(function (err) {
    alert('[from ready][get-app-info][error] ' + JSON.stringify(err, null, 2));
  });

  // 带参数传递调用示例
  lib.exec('get-user-info', {
    params: {
      isRefresh: '1',
    },
  }).then(function (info) {
    alert('[from ready][get-user-info][then] ' + JSON.stringify(info, null, 2));
  }).catch(function (err) {
    alert('[from ready][get-user-info][error] ' + JSON.stringify(err, null, 2));
  });
}, false);
</script>

    加载完成后使用

    加载完成后，同样有两种使用方法

    1. 同加载完成前，使用全局数组，往全局数组 libSQ 中 push 命令和参数;
    2. 直接调用 libSQ 的 exec 方法，传入命令名和对应的参数。
    <script src="./dist/lib-shuqi-js-ios.*.js"></script>
<script>
// 同加载完成前，使用全局数组，往全局数组 `libSQ` 中 `push` 命令和参数;

// 直接调用 `libSQ` 的 `exec` 方法，传入命令名和对应的参数。

const {libSQ: lib} = window;

// 无参数直接调用示例
lib.exec('get-app-info').then(function (info) {
  alert('[from ready][get-app-info][then] ' + JSON.stringify(info, null, 2));
}).catch(function (err) {
  alert('[from ready][get-app-info][error] ' + JSON.stringify(err, null, 2));
});

// 带参数传递调用示例
lib.exec('get-user-info', {
  params: {
    isRefresh: '1',
  },
}).then(function (info) {
  alert('[from ready][get-user-info][then] ' + JSON.stringify(info, null, 2));
}).catch(function (err) {
  alert('[from ready][get-user-info][error] ' + JSON.stringify(err, null, 2));
});
</script>

    高级玩法（处理回调）

    背景 WindVane 的标准事件机制，可能通过在 document 上 addEventListener 来接收来自客户端的消息通知。

    与 iOS 客户端协议，使用 clientEvent 作为客户端通知网页的唯一消息事件，数据格式如下：

    // 格式示例
document.addEventListener('clientEvent', function(evt) {
  const message = event.param;
  const {
    event: eventName,
    data: eventData,
  } = message;
});

    JS SDK 将基于此，对数据进行进一步封装，将 eventData 以 eventName 原生事件，从 document 分发出去，各个业务方自行监听所需事件。

    同时，并不是所有的桥都将回调变更成事件机制，所以 JS SDK 在对应的 JS 桥调用时，会自行进行封装，将之前传入回调的方式统一变更成 document 事件通知的机制。具体事件名称见各 API 的文档。

    JS SDK 转发后的事件，使用 CustomEvent，同时 eventData 会挂载在事件对象的 detail 中。如下示例：

    // 带参数传递调用示例
window.libSQ.exec('get-user-info', {
  params: {
    isRefresh: '1', // 刷新用户信息后通知
  },
}).then(function (info) {
  // 当前的用户信息（刷新前）
  alert('[from ready][get-user-info][then] ' + JSON.stringify(info, null, 2));
}).catch(function (err) {
  alert('[from ready][get-user-info][error] ' + JSON.stringify(err, null, 2));
});

// clientEvent 分发示例 - 获取用户信息
document.addEventListener('getUserInfo', (evt) => {
  // evt.detail 里为刷新之后新获取的用户信息
  console.log('[from getUserInfo] evt: ', evt.detail);
  alert('[from getUserInfo] evt: ' + JSON.stringify(evt.detail, null, 2));
});
